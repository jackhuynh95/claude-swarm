# Phase 1: Comment Sanitizer

**Priority**: Critical
**Status**: Complete
**Roadmap tasks**: #30 (sensitive data filter), #31 (response truncation), #32 (AI disclaimer)

## Overview

Single module that sanitizes all outbound GitHub comments: strip secrets, enforce length limits, prepend AI disclaimer. Every `addComment()` call passes through this pipeline.

## Context Links

- `src/commands/watch/phases/label-manager.ts` — current `addComment()` implementation
- `src/commands/watch/types.ts` — shared types

## Key Insights

- GitHub comment body max: 65,536 characters
- Secrets appear in Claude output when it reads `.env` files or logs
- AI disclaimer required for transparency (GitHub ToS compliance)
- Must be zero-config — patterns hardcoded, not user-configurable

## Architecture

```
sanitizeComment(body: string): string
  1. stripSecrets(body)     — regex redaction
  2. truncateForGithub(body) — 65536 char cap with truncation marker
  3. addDisclaimer(body)     — prepend disclaimer header
```

## Implementation Steps

1. Create `comment-sanitizer.ts` with three exported functions:

2. **`stripSecrets(text: string): string`**
   - Match common secret patterns via regex array:
     - AWS keys: `AKIA[0-9A-Z]{16}`
     - Generic API keys: `(api[_-]?key|apikey|secret|token|password|passwd|auth)\s*[:=]\s*['"]?[^\s'"]{8,}`
     - GitHub tokens: `gh[pousr]_[A-Za-z0-9_]{36,}`
     - npm tokens: `npm_[A-Za-z0-9]{36,}`
     - Base64-encoded secrets: `-----BEGIN (RSA |EC |DSA )?PRIVATE KEY-----`
     - .env variable values: lines matching `^[A-Z_]+=.+` where key contains SECRET/KEY/TOKEN/PASSWORD/AUTH
   - Replace matched values with `[REDACTED]`
   - Return cleaned text

3. **`truncateForGithub(text: string, maxLen?: number): string`**
   - Default `maxLen = 60000` (leave room for disclaimer + formatting)
   - If `text.length > maxLen`: truncate at last newline before limit, append `\n\n---\n*[Output truncated — exceeded GitHub comment limit]*`
   - Return text as-is if within limit

4. **`addDisclaimer(text: string): string`**
   - Prepend: `> **Note**: This comment was generated by an AI assistant (claude-swarm). Review carefully before acting on it.\n\n`
   - Return prefixed text

5. **`sanitizeComment(text: string): string`** — pipeline wrapper:
   ```ts
   export function sanitizeComment(text: string): string {
     return addDisclaimer(truncateForGithub(stripSecrets(text)));
   }
   ```

6. Modify `label-manager.ts` `addComment()`: pass `body` through `sanitizeComment()` before sending to `gh`.

## Risk Assessment

- **False positive redaction**: Regex may match non-secret strings → acceptable tradeoff for safety; keep patterns conservative
- **Disclaimer noise**: Every comment gets disclaimer → intended behavior for transparency
